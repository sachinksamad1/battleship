# ADR 001: Selection of Svelte-Native Libraries

## Status
Accepted

## Context
The project was originally conceived with several React-specific libraries in the stack (e.g., `dnd-kit`, `Framer Motion`, `Zustand`). However, the primary frontend framework is SvelteKit.

## Decision
We will remove all React-specific libraries and replace them with Svelte-native or framework-agnostic alternatives.

- **Drag & Drop**: Replace `dnd-kit` with `svelte-dnd-action`.
- **Animation**: Replace `Framer Motion` with native Svelte transitions (`fade`, `scale`, `slide`, `flip`).
- **State Management**: Replace `Zustand` with native Svelte stores (`writable`, `readable`, `derived`).

## Consequences
- **Positive**: Reduced bundle size, better integration with Svelte's reactivity model, simplified codebase without "wrapper" libraries.
- **Negative**: Developers familiar with the React ecosystem may need a short learning curve for Svelte-native patterns.
