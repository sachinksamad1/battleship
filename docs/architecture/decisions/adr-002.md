# ADR 002: Server-Authoritative Multiplayer Model

## Status
Accepted

## Context
Multiplayer games are susceptible to client-side cheating (e.g., viewing hidden ship locations, firing multiple times per turn).

## Decision
We will implement a **Server-Authoritative** model for all multiplayer matches.

- The server maintains the master state of both boards.
- Clients send "Intents" (e.g., "I want to attack A1").
- The server validates the intent (Is it the player's turn? Is the coordinate valid?).
- Only the server calculates hit/miss/sink results.
- The server only sends the *necessary* state updates back to the client (e.g., hiding the full enemy board until the end of the game).

## Consequences
- **Positive**: High security, consistent state between players, easier reconnection handling.
- **Negative**: Increased server-side CPU and memory usage; slightly higher perceived latency for attack feedback compared to client-side prediction.
